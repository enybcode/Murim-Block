# Reference Analysis: Sorcery Age Showcase

Source watched: https://www.youtube.com/watch?v=mTwLMlSfxJI

Video title observed: `NEW MOD | Sorcery Age! | Limitless, Shrine, Ten Shadows... | Minecraft Jujutsu Kaisen Mod`

## What Makes It Feel Strong

- The fantasy is immediate: the thumbnail and intro communicate powers before the player reads anything.
- Techniques are treated like identity, not just damage buttons: Limitless, Shrine, Ten Shadows style abilities each have a visual language.
- The player has many keybinds, which makes the mod feel like a real combat system instead of a few items.
- The HUD/chat feedback explains state changes, technique choices, energy/cursed energy, and progression.
- The showcase uses a clean city arena, making particles, mobs and movement easier to read.
- The mod sells scale through large silhouettes: summons, portals/domains, oversized attacks, and strong color contrast.

## Murim Block Translation

We should not copy JJK assets or mechanics directly. The target is the same quality level with Murim flavor.

- `Limitless` style readability becomes qi barriers, sword intent fields, body flicker, palm pressure, killing intent.
- `Shrine` style area danger becomes sword domains, formation circles, slash marks, sect forbidden arts.
- `Ten Shadows` style summon identity becomes spirit beasts, martial avatars, qi constructs, weapon spirits.
- Cursed energy HUD becomes qi / dantian / realm / breakthrough pressure HUD.
- Technique menu becomes martial manual / technique tree.
- Urban JJK arena becomes sect courtyard, training ground, bamboo forest, demon cave, tournament stage.

## Production Rules

- Every major ability needs a 3-part visual rhythm: charge, release, aftermath.
- Every unlock needs ceremony: advancement, GUI change, sound, particle or chat line.
- Every combat feature needs readable state: locked, ready, active, cooldown, insufficient qi.
- Every mob needs a purpose in cultivation: training enemy, beast core source, sect trial, boss, summon.
- Every asset should be traceable from idea -> Blockbench/texture -> MCreator element -> test -> commit.

## First Concrete Targets

- Fix and polish the technique GUI so it feels like a real martial manual.
- Upgrade First Sword Art into a signature first wow moment.
- Build a repeatable technique template: keybind, qi cost, cooldown, particles, damage/effect, unlock state.
- Build a repeatable mob template: Blockbench model, texture, MCreator entity, drops, test arena.
- Create a small test arena/structure for showcasing abilities cleanly.
